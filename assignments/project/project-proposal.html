Prepare a written proposal for your project. The project should provide some useful functionality in science or engineering. It could be a command line utility, or a package to use in notebooks. There should be some substance, but it does not need to be extensive. I don‚Äôt expect it should take more than a few hours to write the code. It is not necessary to write very sophisticated code. Eventually the project must be installable, and it should use version control, at least one code quality tool, and must have tests that pass. Overall the project should demonstrate you have learned something in this class.

I suggest that you write some kind of package around OpenAlex, but if you have another idea you are welcome to propose it. The project should be relevant to scientific problems (e.g. I don‚Äôt want proposals to help select restaurants, or other non-scientific tasks).

Your proposal should identify the problem you are trying to solve, provide some ideas on how you will solve it, and include an estimate of how long you think it will take. You should make sure you can complete the project in about a week.
